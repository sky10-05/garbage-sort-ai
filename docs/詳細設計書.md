# 詳細設計書

## システム名

自治体対応型AIごみ分別支援システム

## 1. 目的

本書は、仮データによる画面遷移実装と、今後のYOLO連携に向けた機能単位の処理内容を定義する。

自治体は利用開始ごとに選択し、選択中の自治体はFlaskセッション内のみ保持する。端末には永続保存しない。

## 2. 共通定数

| 定数 | 値 | 内容 |
|---|---:|---|
| `CONFIDENCE_THRESHOLD` | `0.70` | 判定成功とする信頼度の下限 |
| `MAX_RETRY_COUNT` | `3` | 再撮影を案内する上限回数 |
| `MAX_CANDIDATES` | `3` | 判定不可時に表示する候補数 |
| `HISTORY_DISPLAY_LIMIT` | `2` | 履歴画面に表示する件数 |
| `MAX_IMAGE_SIZE` | `5MB` | アップロード画像の最大サイズ |

## 5. 仮判定結果

YOLO連携前は以下の仮結果を利用する。

```python
result = {
    "label": "plastic_bottle",
    "display_name": "ペットボトル",
    "confidence": 0.92
}
```

信頼度は画面またはフォーム値で変更でき、0.70以上なら判定結果、0.70未満なら再撮影案内へ分岐する。

## 6. エラー方針

利用者向けには短い原因と次の操作を表示する。開発者向けには `app.logger` へ詳細を出力する。

対象エラー:

- 自治体未選択
- カメラ利用拒否
- 画像未送信
- 対応していない画像形式
- ファイルサイズ超過
- DB接続エラー
- DBに存在しないAIラベル
- 分別ルール未登録
- 質問データ未登録
- 履歴保存失敗
- feedback保存失敗
- 音声再生失敗
- 一時画像削除失敗

## 7. テスト仕様

| No | 確認内容 | 期待結果 |
|---|---|---|
| 1 | 自治体一覧を取得できる | 自治体選択画面に自治体が表示される |
| 2 | 自治体を選択できる | 撮影画面へ遷移する |
| 3 | 自治体未選択では撮影画面へ進めない | 自治体選択画面へ戻る |
| 4 | 信頼度0.70で正常判定になる | 判定結果画面へ進む |
| 5 | 信頼度0.69で再撮影になる | 再撮影案内を表示する |
| 6 | 再撮影上限後に候補3件が表示される | 候補一覧を表示する |
| 7 | AIラベルから正しいごみ情報を取得できる | ペットボトルを取得する |
| 8 | 自治体別の分別ルールを取得できる | 資源ごみのルールを取得する |
| 9 | 存在しないAIラベルでエラー処理される | エラー表示とログ出力を行う |
| 10 | 履歴が正常に保存される | `history` に登録される |
| 11 | 履歴画面に最新2件のみ表示される | 2件だけ表示される |
| 12 | feedbackが保存される | `feedback` に登録される |
| 13 | 音声OFFで読み上げされない | Web Speech APIを呼ばない |
| 14 | 保存未同意の画像が削除される | 一時画像が残らない |
| 15 | 不正な画像形式が拒否される | エラー表示する |
| 16 | SQLインジェクションを試す | プレースホルダにより不正SQLが実行されない |

## 8. YOLO連携時の変更箇所

- `app.py` の `get_mock_result()` を `ai/model.py` の推論呼び出しへ置き換える。
- 信頼度とラベルの返却形式は本書の仮判定結果と同じ形式に揃える。
- 画像保存・削除・履歴保存の処理はそのまま利用する。
