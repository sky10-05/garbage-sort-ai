# YOLOモデル評価改善

## 目的

YOLO学習済みモデルの評価結果と実機確認で見つかった誤判定を記録し、次回のデータセット改善と再学習につなげる。

## 対象モデル

| 項目 | 内容 |
|---|---|
| 学習元モデル | `yolo11n.pt` |
| 学習済みモデル | `runs/detect/train-11/weights/best.pt` |
| 学習データ | `data/yolo_dataset/data.yaml` |
| 学習回数 | 30 epoch |
| 画像サイズ | 640 |
| バッチサイズ | 4 |
| 実行デバイス | CUDA:0 |

学習済み重みファイルは容量と運用方針を確認するまでGit管理対象外とする。

## 評価メモ

- `dry_battery` と `aerosol_can` はmAP50が高く、現時点では比較的強い。
- `plastic_bottle`、`drink_can`、`glass_bottle` は精度が低く、追加データまたはラベル品質の確認が必要である。
- `glass_bottle` はRecallが0.000であり、評価データ上ではほぼ検出できていない。
- `aerosol_can` と `dry_battery` が強いため、弱いクラスや背景をこれらに誤判定していないか確認する。

## 改善方針

### plastic_bottle

- 透明なペットボトル、ラベル付き、キャップ付き、つぶれたものを増やす。
- スプレー缶と形が似ている画像を重点的に確認し、誤って `aerosol_can` に寄らないようにする。
- 背景が生活環境に近い画像を増やす。

### drink_can

- アルミ缶、スチール缶、色の違う飲料缶を増やす。
- 横向き、縦向き、つぶれた缶を分けて確認する。
- スプレー缶や缶詰と混ざらないよう、クラス定義を明確にする。

### glass_bottle

- 最優先で追加データを集める。
- 透明びん、茶色びん、緑びん、ラベル付きびんを増やす。
- 評価データのラベル位置とクラスIDを再確認する。
- 手元に実物がない場合は、ライセンス確認済みの公開データや自前撮影で補強する。

### aerosol_can

- 現時点では強いが、ペットボトルを誤って吸っている可能性がある。
- スプレー缶らしい金属容器だけを残せているか再確認する。
- トリガーボトル、ポンプ式ボトル、通常飲料缶との境界を確認する。

### dry_battery

- 現時点では強いが、顔や背景を乾電池と誤判定するケースがある。
- 背景・負例データの不足を確認する。
- ボタン電池、9V角形電池、バッテリーパックを混ぜない方針は維持する。

## 次回学習前に確認すること

- 追加データのライセンスを確認する。
- 画像に個人情報や不要な写り込みがないか確認する。
- `plastic_bottle` と `glass_bottle` は追加データセットで補強済みのため、再学習後の改善幅を確認する。
- `drink_can` はまだ弱いため、必要に応じて次の補強対象にする。
- 背景・判定対象外画像をどう扱うか検討する。
- 同じ画像や類似画像がtrain/val/testをまたいで混ざらないようにする。

## 次回学習コマンド案

データ改善後、まずは30epochで再評価する。

```bash
python3 ai/train.py \
  --model yolo11n.pt \
  --epochs 30 \
  --imgsz 640 \
  --batch 4 \
  --workers 0 \
  --no-amp \
  --device 0
```
