# 画面設計書

## システム名

自治体対応型AIごみ分別支援システム

## 2. 目的

本書は、自治体対応型AIごみ分別支援システムにおける画面構成、画面遷移、各画面の表示項目、操作内容、エラー表示方針を定義するものである。

利用者がカメラでごみを撮影し、AI画像認識と自治体別分別ルールDBをもとに、正しい分別区分と捨て方を迷わず確認できる画面を設計する。

本書は、要件定義書、基本設計書、DB設計書と整合するように作成し、HTML、CSS、JavaScript、Python、Flaskによる実装時の画面仕様として利用する。

## 3. システム概要

本システムは、Webブラウザ上で利用するAIごみ分別案内システムである。利用者が自治体を選択し、カメラでごみを撮影すると、YOLOによる画像認識結果とSQLiteに登録された自治体ごとの分別ルールを照合し、画面表示と音声案内で捨て方を案内する。

初回起動時のみ自治体を選択し、選択した自治体はローカル環境に保存する。次回以降はトップ画面から開始し、トップ画面から自治体を変更することもできる。

AIの信頼度が十分な場合は、自治体別ルールを確認して判定結果画面へ進む。分別結果の確定に条件分岐質問が必要な場合は質問画面で選択式質問を表示し、回答内容をもとに最終判定を行う。信頼度が低い場合、認識できない場合、未対応ごみの場合、自治体ルールが未登録の場合、またはカメラエラーが発生した場合はエラー画面を表示し、再撮影や確認を促す。


#### 画面項目一覧

| 項目名 | 種別 | 表示内容・入力内容 | 必須 | 備考 |
|---|---|---|---|---|

| 都道府県選択 | セレクトボックス | `prefecture.prefecture_name` | 必須 | DBから取得 |
| 市区町村選択 | セレクトボックス | `municipality.municipality_name` | 必須 | 選択した都道府県に紐づく市区町村を表示 |

#### 処理内容

1. 画面表示時に都道府県一覧をDBから取得する。
2. 利用者が都道府県を選択すると、該当する市区町村一覧を表示する。
3. 利用者が市区町村を選択し、「この自治体で始める」を押す。
4. 選択した `municipality_id` をローカル環境に保存する。
5. トップ画面へ遷移する。

#### エラーメッセージ

| 状況 | メッセージ |
|---|---|
| 都道府県未選択 | 都道府県を選択してください。 |
| 市区町村未選択 | 市区町村を選択してください。 |
| 自治体一覧取得失敗 | 自治体情報を読み込めませんでした。時間をおいてもう一度お試しください。 |

#### 備考

- 初回起動時のみ表示する。
- 保存方式は実装時に `localStorage`、Flaskセッション、またはローカル設定ファイルから選定する。
- 利用者個人を管理するDBテーブルは初期版では作成しない。

### 6.3 SCR-03 認識画面

| 項目 | 内容 |

| 画面ID | SCR-03 |
| 画面名 | 認識画面 |
| 画面概要 | カメラ映像を表示し、ごみを撮影してAI画像認識を行う画面。 |
| 遷移元 | トップ画面、判定結果画面、エラー画面 |
| 遷移先 | SCR-05 判定結果画面、SCR-04 質問画面、SCR-07 エラー画面、SCR-02 トップ画面 |

```text
│   ┌──────────────────┐     │
│   │                  │     │
│   │                  │     │
│   │                  │     │
│   └──────────────────┘     │
│                            │
│ [ ] AI改善のため、撮影画像の │
│     保存に同意します         │
│                            │
│        [ 撮影する ]         │
│        [ トップへ ]         │
│                            │

└────────────────────────────┘
```

#### 画面項目一覧

| 項目名 | 種別 | 表示内容・入力内容 | 必須 | 備考 |
|---|---|---|---|---|
| 画面タイトル | テキスト | AIごみ分別案内システム | - | 全画面で統一 |
| 自治体名 | テキスト | 現在選択中の市区町村名 | - | 認識時のルール検索に使用 |
| カメラ映像 | 映像表示 | Webカメラ映像 | - | ブラウザ上に表示 |
| 画像保存同意 | チェックボックス | AI改善のため、撮影画像の保存に同意します | 任意 | 同意しない場合でも判定可能 |
| 撮影するボタン | ボタン | 画像を撮影してAI認識を開始 | - | 主要操作として大きく配置 |
| トップへボタン | ボタン | トップ画面へ戻る | - | 判定を中止する場合に使用 |
| 注意文 | テキスト | ごみを1つだけ映してください。 | - | 要件の「1回につき1つ」を明示 |
| 処理中表示 | テキスト | 判定中です。しばらくお待ちください。 | - | AI認識中に表示 |

#### 処理内容

1. カメラ使用許可を確認する。
2. カメラ映像を画面に表示する。
3. 利用者がごみを1つだけ映す。
4. 必要に応じて、利用者が画像保存同意チェックボックスを選択する。
5. 「撮影する」を押すと画像を取得し、バックエンドへ送信する。
6. YOLOモデルで画像を解析し、ごみ種類と信頼度を取得する。
7. 信頼度が十分な場合は、自治体IDとごみ種類をもとにDBから分別ルールを取得する。
8. 分別ルールに条件分岐質問が不要な場合は、判定結果画面へ遷移する。
9. 分別ルールに条件分岐質問が必要な場合は、質問画面へ遷移する。
10. 信頼度が低い場合、認識できない場合、未対応ごみの場合、分別ルールが見つからない場合、またはカメラエラーの場合はエラー画面へ遷移する。

#### エラーメッセージ

| 状況 | メッセージ |
|---|---|
| カメラ使用不可 | カメラを使用できません。接続やブラウザの許可設定を確認してください。 |
| 撮影失敗 | 画像を撮影できませんでした。もう一度お試しください。 |
| 通信失敗 | 判定処理に接続できませんでした。時間をおいてもう一度お試しください。 |
| 複数ごみ検出 | ごみは1つだけ映してください。 |

#### 備考

- 信頼度の閾値は実装時に設定値として変更できるようにする。
- 画像は、利用者が画像保存に同意した場合のみ履歴または改善用データとして保存する。
- 画像保存に同意しない場合でも、AI判定、分別ルール検索、判定結果表示は行える。

| 項目 | 内容 |

| 画面名 | 質問画面 |
| 画面概要 | 分別結果の確定に条件分岐質問が必要な場合に、質問を1問ずつ表示する画面。 |
| 遷移元 | 認識画面 |
| 遷移先 | SCR-05 判定結果画面、SCR-07 エラー画面、SCR-03 認識画面、SCR-02 トップ画面 |

#### ワイヤーフレーム

```text
┌────────────────────────────┐
│ AIごみ分別案内システム       │
│ 自治体：名古屋市             │
├────────────────────────────┤
│ 判定のために確認します。       │
│                            │
│ 質問 1 / 2                  │
│ このごみは汚れていますか？     │
│                            │
│        [ はい ]             │
│        [ いいえ ]           │
│        [ わからない ]        │
│                            │
│ [ もう一度撮影 ] [ トップへ ] │
└────────────────────────────┘
```

#### 画面項目一覧

| 項目名 | 種別 | 表示内容・入力内容 | 必須 | 備考 |
|---|---|---|---|---|
| 画面タイトル | テキスト | AIごみ分別案内システム | - | 全画面で統一 |
| 自治体名 | テキスト | 現在選択中の市区町村名 | - | 判定対象の自治体 |
| 案内文 | テキスト | 判定のために確認します。 | - | 不安を与えない表現にする |
| 質問番号 | テキスト | 質問 1 / 2 | - | 最大2問まで |
| 質問文 | テキスト | `question_type.question_text` | - | DBから取得 |
| 回答ボタン | ボタン | はい、いいえ、わからない等 | 必須 | `question_answer.answer_value` をもとに表示 |
| もう一度撮影ボタン | ボタン | 認識画面へ戻る | - | 判定をやり直す |
| トップへボタン | ボタン | トップ画面へ戻る | - | 判定を中止する |

#### 処理内容

1. 認識画面からごみ種類、信頼度、撮影画像情報、画像保存同意の有無を受け取る。
2. 自治体IDとごみ種類をもとに、条件分岐質問をDBから取得する。
3. 質問を1問ずつ表示する。
4. 利用者が「はい」「いいえ」「わからない」などの回答を選択する。
5. 回答内容に応じて次の質問を表示する、または最終判定を確定する。
6. 最終判定が確定した場合は判定結果画面へ遷移する。
7. 判定できない場合はエラー画面へ遷移する。
8. 質問と回答内容は、判定履歴と紐づけて保存できるようにする。

#### エラーメッセージ

| 状況 | メッセージ |
|---|---|
| 質問取得失敗 | 確認用の質問を読み込めませんでした。もう一度判定してください。 |
| 回答未選択 | 回答を選択してください。 |
| 判定不能 | 回答内容から分別方法を決定できませんでした。 |

#### 備考

- DB設計書の `question_type`、`question_answer`、`history_question_answer` と対応する。
- 質問数は最大2問までとする。
- 自由入力ではなく選択式とする。

| 信頼度 | テキスト | 0〜100% | - | `history.confidence` を百分率表示 |
| 分別区分 | テキスト | `garbage_type.type_name` | - | 例: 可燃ごみ、資源ごみ、有害ごみ |
| 捨て方 | 番号付きリスト | `garbage_rule.guide_text` または回答後案内文 | - | 手順として表示 |
| 注意事項 | テキスト | `garbage_rule.note` | - | 登録がある場合のみ表示 |
| 音声案内状態 | テキスト | 音声案内中、音声OFF、音声案内完了 | - | 画面表示と音声内容を一致させる |
| 追加質問ボタン | ボタン | 注意事項はありますか？、いつ出せますか？、どこに出せますか？ | - | `info_question_type` から表示 |
| フィードバック | ボタン | はい、ふつう、いいえ | - | 初期版では画面表示のみ。保存は将来拡張 |
| もう一度判定ボタン | ボタン | 認識画面へ遷移 | - | 再判定 |
| トップへボタン | ボタン | トップ画面へ遷移 | - | 判定終了 |

#### 処理内容

1. AI認識結果、質問回答結果、自治体別分別ルールを受け取る。
2. ごみ名、信頼度、認識画像、分別区分、捨て方、注意事項を表示する。
3. 捨て方は自治体ごとに異なるため、DBから取得した案内文を表示する。
4. 捨て方が複数手順で表現できる場合は番号付きリストとして表示する。
5. 音声案内ONの場合は、画面表示と同じ内容をPCスピーカーから読み上げる。
6. 音声案内中は「音声案内中」と表示する。
7. 判定結果を `history` に保存する。
8. 質問画面を経由した場合は、質問回答内容を `history_question_answer` に保存する。
9. 利用者が追加質問ボタンを選択した場合、`info_question_type` と `info_answer` をもとに補足情報を取得し、画面表示と音声案内を行う。
10. 認識画像は画面上で一時表示する。画像保存同意がない場合は、ファイル保存や履歴への画像パス保存は行わない。
11. 「もう一度判定」を押すと認識画面へ遷移する。
12. 「トップへ」を押すとトップ画面へ遷移する。

#### エラーメッセージ

| 状況 | メッセージ |
|---|---|
| 判定結果取得失敗 | 判定結果を表示できませんでした。もう一度判定してください。 |
| 音声案内失敗 | 音声案内を再生できませんでした。画面の内容を確認してください。 |
| 履歴保存失敗 | 判定履歴を保存できませんでした。判定結果は画面で確認できます。 |

#### 備考

- 追加情報質問として「注意事項」「いつ出せるか」「どこに出せるか」などを表示する場合は、DB設計書の `info_question_type`、`info_answer` を利用する。
- フィードバックは本画面では「はい」「ふつう」「いいえ」の3段階とし、詳細保存は将来拡張とする。
- フィードバック保存を実装する場合は、DB設計書で評価用テーブルを追加してから対応する。

### 6.7 SCR-07 エラー画面

| 項目 | 内容 |
|---|---|
| 画面ID | SCR-07 |
| 画面名 | エラー画面 |

```text
│                            │
│        [ もう一度判定 ]       │
│        [ トップへ ]           │
└────────────────────────────┘
```

#### 画面項目一覧

| 項目名 | 種別 | 表示内容・入力内容 | 必須 | 備考 |
|---|---|---|---|---|
| 画面タイトル | テキスト | AIごみ分別案内システム | - | 全画面で統一 |
| エラー見出し | テキスト | 判定できませんでした。 | - | 分かりやすく表示 |
| 原因 | テキスト | 認識失敗、未対応ごみ、ルール未登録など | - | エラー種別に応じて変更 |

#### 処理内容

1. エラー種別、信頼度、候補取得状況を受け取る。
2. エラー種別に応じた原因と対処方法を表示する。
3. 認識できない場合、未対応ごみの場合、または自治体ルールが未登録の場合は、該当する理由を表示する。
4. 必要に応じて `history.result_status` に `unknown`、`low_confidence`、`no_rule` などを保存する。
5. 「もう一度判定」を押すと認識画面へ遷移する。
6. 「トップへ」を押すとトップ画面へ遷移する。

#### エラーメッセージ

| 状況 | メッセージ | 対処方法 |
|---|---|---|
| 認識できない | ごみをはっきり認識できませんでした。 | 明るい場所で、ごみを1つだけ映してもう一度撮影してください。 |
| 信頼度が低い | 判定に必要な情報が不足しています。 | 角度や距離を変えてもう一度撮影してください。 |
| 未対応ごみ | このごみは現在対応していません。 | 今後の対応追加まで、自治体の案内をご確認ください。 |
| ルール未登録 | 選択中の自治体では、このごみの分別ルールが登録されていません。 | 自治体の公式案内をご確認ください。 |
| カメラエラー | カメラを使用できません。 | 接続やブラウザの許可設定を確認してください。 |
| 通信エラー | 判定処理に接続できませんでした。 | 時間をおいてもう一度お試しください。 |

#### 備考

- エラー時は原因と対処方法を必ずセットで表示する。
- 未対応ごみや判定不可は、将来的にフィードバック登録や学習データ追加に活用できるようにする。

## 7. 共通画面ルール

| 項目 | ルール |
|---|---|
| 画面構成 | シンプルで見やすい構成にする |
| 文字サイズ | 高齢者にも読みやすい大きめの文字を使用する |
| ボタン | 主要ボタンは大きめにし、押しやすい余白を確保する |
| 重要情報 | 判定結果、分別区分、捨て方などは画面中央付近に配置する |
| 音声案内 | 画面表示と音声案内の内容を一致させる |
| エラー表示 | 原因と対処方法を必ず表示する |
| 入力方式 | 可能な限り選択式にし、自由入力は最小限にする |
| 表記 | 「ごみ」「自治体」「分別区分」「捨て方」の用語を統一する |

## 8. デザイン方針

高齢者を含む幅広い利用者が迷わず操作できるUIとする。

画面ごとの主操作を明確にし、重要なボタンは大きく配置する。判定結果画面では認識画像を表示し、AIが何を認識したか利用者が確認できるようにする。分別区分と捨て方は、自治体ごとのDBから取得した内容をもとに表示し、番号付き手順で読みやすく整理する。

色や装飾は控えめにし、判定結果、注意事項、エラー内容などの重要情報が埋もれないようにする。

## 9. アクセシビリティ

| 項目 | 方針 |
|---|---|
| 文字サイズ | 大きな文字を使用し、読みやすさを優先する |
| 日本語表現 | 専門用語を避け、分かりやすい短文にする |
| 音声案内 | 画面表示と同じ内容を音声でも案内する |
| ボタン | 大きく押しやすいボタンにする |
| 情報量 | 1画面に詰め込みすぎず、必要最低限にする |
| エラー案内 | 原因だけでなく、次に行う操作を明示する |
