# 基本設計書

## システム名

自治体対応型AIごみ分別支援システム

## 1. システム概要

本システムは、利用者がカメラの前に置いたごみをAI画像認識によって判定し、自治体ごとの分別ルールに基づいて、適切な捨て方を画面表示と音声で案内するWebシステムである。

また、判定後に選択式の質問機能を用意し、利用者が「注意事項」「出す場所」「出す日」などを追加で確認できる対話型マルチモーダルシステムとして構成する。

本システムはローカルPCでの実行を前提とし、Webブラウザから操作する。

## 2. 開発目的

自治体によってごみの分別ルールは異なるため、利用者が正しい捨て方を判断できない場合がある。

特に、電池、モバイルバッテリー、スプレー缶、ライター、蛍光灯などは、捨て方を誤ると火災や事故につながる可能性がある。

本システムでは、AI画像認識と自治体別ルールDBを組み合わせることで、利用者が迷いやすいごみを正しく分別できるよう支援する。

## 3. システムの特徴

- Webブラウザ上で利用できる
- ローカルPC上でFlaskアプリとして実行する
- カメラで撮影したごみをAIで判定する
- 自治体ごとの分別ルールをDBで管理する
- PythonからPCスピーカーへ音声案内を出力する
- 追加質問は自由入力ではなく選択式で行う
- 一般ごみに加えて、捨て方に迷いやすいごみを重点的に扱う
- 判定履歴を保存し、後から参照できるようにする

## 4. 対象ユーザー

- 一般家庭
- 高齢者
- 引っ越してきた住民
- 外国人住民
- 公共施設利用者
- ごみの捨て方に迷う利用者

## 7. 画面設計

### 7.1 自治体選択画面

利用者が利用する自治体を選択する画面である。

自治体選択は、都道府県を選択したあと、市区町村を選択する2段階方式とする。

選択した自治体は毎回リセットされ、次回利用時に自動保存はしない。

| 項目 | 内容 |
|---|---|
| 都道府県選択 | 都道府県を選択する |
| 市区町村選択 | 選択した都道府県に属する市区町村を選択する |
| 決定ボタン | 選択した自治体でメイン画面へ進む |

### 7.2 メイン画面

ごみを撮影し、判定結果を確認する画面である。

| 項目 | 内容 |
|---|---|
| 選択中の自治体 | 現在選択している自治体名を表示する |
| カメラ映像 | Webカメラの映像を表示する |
| 撮影ボタン | カメラ画像を撮影し、AI判定を開始する |
| 音声ON/OFF | 音声案内の有効・無効を切り替える |
| 判定結果 | AIが判定したごみの種類を表示する |
| 信頼度 | AI判定の信頼度を表示する |
| 分別方法 | 自治体ルールに基づく分別区分を表示する |
| 捨て方 | 具体的な捨て方を表示する |
| 注意事項 | 捨てる際の注意点を表示する |
| 追加質問 | 利用者が知りたい内容を選択する |

### 7.3 履歴一覧画面

過去に判定したごみと分別結果を参照する画面である。

| 項目 | 内容 |
|---|---|
| 判定日時 | 判定した日時を表示する |
| 自治体 | 判定時に選択していた自治体を表示する |
| ごみ種類 | AIが判定したごみ種類を表示する |
| 分別区分 | 判定時の分別区分を表示する |
| 信頼度 | 判定時の信頼度を表示する |

## 8. 機能設計

### 8.1 自治体選択機能

利用者が分別ルールを適用する自治体を選択する。

入力:

- 都道府県
- 市区町村

出力:

- 自治体ID

処理内容:

1. 都道府県一覧を表示する。
2. 選択された都道府県に対応する市区町村一覧を表示する。
3. 利用者が市区町村を選択する。
4. 選択された自治体IDを判定処理で使用する。
5. 自治体情報は次回利用時に保存しない。

### 8.2 カメラ撮影機能

Web画面上でカメラ映像を表示し、撮影ボタンでごみ画像を取得する。

入力:

- Webカメラ映像

出力:

- 撮影画像

処理内容:

1. Webブラウザ上でカメラを起動する。
2. 利用者がごみをカメラ前に置く。
3. 利用者が撮影ボタンを押す。
4. 撮影画像をバックエンドへ送信する。

### 8.3 AI画像認識機能

撮影画像をYOLOモデルで解析し、ごみの種類を判定する。

入力:

- 撮影画像

出力:

- ごみ種類
- 信頼度

処理内容:

1. 撮影画像を受け取る。
2. YOLOモデルで画像を解析する。
3. ごみの種類と信頼度を取得する。
4. 信頼度が一定値以上であれば判定結果として使用する。
5. 信頼度が低い場合は、再撮影を促す。

補足:

- AIモデルは既存モデルの利用だけでなく、自作データセットによる追加学習も行う。
- GitHub Publicリポジトリには学習画像や公開データセット本体をコミットしない。
- `data/yolo_dataset/` には構成管理用の `.gitkeep` のみ配置する。

### 8.4 分別ルール検索機能

AIが判定したごみ種類と選択された自治体をもとに、SQLiteから分別ルールを取得する。

入力:

- 自治体ID
- ごみ種類

出力:

- 分別区分
- 捨て方
- 注意事項
- 出す日
- 出す場所
- 禁止事項

処理内容:

1. 自治体IDとごみ種類を受け取る。
2. 分別ルールDBを検索する。
3. 該当する分別ルールを取得する。
4. 判定結果表示機能へ渡す。

### 8.5 判定結果表示機能

AI判定結果と自治体ごとの分別ルールを画面に表示する。

表示内容:

- ごみの種類
- 信頼度
- 分別区分
- 捨て方
- 注意事項

処理内容:

1. バックエンドから判定結果を受け取る。
2. 画面の判定結果欄に表示する。
3. 判定できない場合は、再撮影を促すメッセージを表示する。

### 8.6 音声案内機能

判定結果と分別方法をPCスピーカーから音声で案内する。

今回はローカル実行を前提とし、音声案内はPythonからPCスピーカーへ出力する。ブラウザ側のWeb Speech APIは採用しない。

入力:

- ごみ種類
- 分別区分
- 捨て方
- 注意事項

出力:

- 音声案内

処理内容:

1. 画面表示用の判定結果をもとに案内文を作成する。
2. 音声ONの場合、PCスピーカーから読み上げる。
3. 音声OFFの場合、読み上げは行わない。

音声例:

```text
これは乾電池です。
岐阜市では有害ごみとして出してください。
端子部分をテープで絶縁してください。
```

### 8.7 選択式質問機能

判定結果表示後、利用者が追加で知りたい内容を選択できる機能である。

自由入力ではなく、あらかじめ用意した質問ボタンを選択する方式とする。

| 質問 | 内容 |
|---|---|
| 注意事項はありますか？ | 捨てる前に必要な準備や注意点を案内する |
| いつ出せますか？ | 収集日や回収日を案内する |
| どこに出せますか？ | 集積所や回収ボックスなどを案内する |
| 他のごみとして捨てられますか？ | 誤った捨て方や禁止事項を案内する |
| もう一度撮影する | 判定をやり直す |

処理内容:

1. 判定結果表示後、追加質問ボタンを表示する。
2. 利用者が質問を選択する。
3. 選択された質問種別に応じてDBから回答を取得する。
4. 回答を画面に表示する。
5. 音声ONの場合は回答を音声で案内する。

### 8.8 判定履歴保存・参照機能

判定したごみと分別結果を履歴として保存し、後から参照できるようにする。

履歴は、判定結果を画面に表示した時点で保存する。追加質問の履歴を保存するかどうかはDB設計時に検討する。

保存項目:

- 判定日時
- 自治体
- ごみ種類
- 分別区分
- 信頼度

処理内容:

1. 判定結果を画面に表示する。
2. 表示した判定結果を履歴としてSQLiteに保存する。
3. 履歴一覧画面で過去の判定結果を表示する。

## 10. AI学習方針

AI画像認識にはYOLOを使用する。

初期段階では少数のごみ種類で学習・検証を行い、システム全体の動作確認を優先する。

| 段階 | 1種類あたりの画像枚数 | 目的 |
|---|---:|---|
| 試作 | 30〜50枚 | 学習できるか確認する |
| 最低ライン | 100枚前後 | デモで使える精度を目指す |
| 目標 | 200〜300枚 | より安定した判定を目指す |

データ収集方法:

- 自分で撮影した画像
- 利用条件を確認した公開データセット
- 必要に応じてインターネット上の参考画像

注意点:

- インターネット上の画像を利用する場合は、著作権や利用条件に注意する。
- GitHub Publicリポジトリには学習画像やデータセット本体をコミットしない。
- データセット構成のみを管理し、画像本体はローカル環境で管理する。

## 11. データベース方針

データベースにはSQLiteを使用する。

DBの詳細なテーブル構成やカラム設計は、別途「DB設計書」で整理する。

基本設計では、以下の情報を管理する方針とする。

| 管理情報 | 内容 |
|---|---|
| 都道府県情報 | 都道府県名を管理する |
| 自治体情報 | 市区町村名を管理する |
| ごみ種類情報 | AIが判定するごみの種類を管理する |
| 分別ルール情報 | 自治体ごとの分別方法を管理する |
| 質問回答情報 | 選択式質問に対する回答を管理する |
| 判定履歴情報 | 過去の判定結果を管理する |

## 12. 処理フロー

```text
1. 利用者が都道府県を選択する
2. 利用者が市区町村を選択する
3. メイン画面でカメラを起動する
4. 利用者がごみをカメラ前に置く
5. 撮影ボタンを押す
6. 撮影画像をバックエンドへ送信する
7. YOLOでごみの種類を判定する
8. 自治体IDとごみ種類をもとにDBを検索する
9. 分別方法・捨て方・注意事項を取得する
10. 判定結果を画面に表示する
11. 音声ONの場合、判定結果を音声案内する
12. 判定履歴を保存する
13. 「さらに知りたいことはありますか？」と表示する
14. 利用者が質問ボタンを選択する
15. 選択された質問に応じてDBから回答を取得する
16. 回答を画面表示・音声案内する
```

## 13. 例外処理

| 状況 | 対応 |
|---|---|
| 自治体が未選択 | 自治体選択画面へ戻す |
| カメラが使用できない | カメラを使用できない旨を表示する |
| ごみを判定できない | 再撮影を促す |
| 信頼度が低い | 判定結果に注意メッセージを表示する |
| DBに分別ルールがない | ルール未登録であることを表示する |
| 音声案内がOFF | 画面表示のみ行う |
| 質問に対応する回答がない | 回答未登録であることを表示する |
